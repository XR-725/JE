# 民宿预约管理系统 -- 前后端启动教程

## 环境要求

| 组件 | 版本 / 说明 |
|------|------------|
| JDK | **17**（默认路径 `C:\Program Files\Java\jdk-17`） |
| Node.js | **16+** 或 **18+**（推荐 18 LTS） |
| MySQL | **8.0+**，端口 `3306` |
| Redis | 端口 `6379`，密码 `204725` |
| Maven | **不需要手动安装**，项目自带 Maven Wrapper |

---

## 第一步：启动基础设施（MySQL + Redis）

### 1.1 启动 MySQL

确保 MySQL 服务在 `localhost:3306` 运行。Windows 下通常：

```powershell
# 检查 MySQL 是否运行
net start | findstr MySQL

# 若未运行，启动服务（管理员终端）
net start MySQL80
```

### 1.2 创建数据库 & 导入初始数据

用你习惯的工具（Navicat / DBeaver / MySQL CLI）执行以下 SQL 脚本：

```
homestay-booking\src\main\resources\init.sql          -- 建表 + 测试数据（8张表）
homestay-booking\src\main\resources\db-migration.sql  -- 系统配置表 + 操作日志表（2张表）
```

**数据库连接信息：**

```
主机:     localhost:3306
数据库:   XRXRXR
用户名:   XRXRXR
密码:     XRXRXR
字符集:   utf8mb4
```

> **注意**：两个脚本执行完后共 **10 张表**，已内置大量测试数据。

### 1.3 启动 Redis

确保 Redis 在 `localhost:6379` 运行，登录密码为 `204725`。

```powershell
# 检查 Redis 端口是否在监听
netstat -ano | findstr :6379

# 若未运行，可进入 Redis 安装目录执行
redis-server.exe redis.windows.conf
```

## 第二步：启动后端（Spring Boot）

### 2.1 确认上传目录

后端文件上传目录配置为：

```
C:\Users\xr\Desktop\000\1\homestay-booking\uploads
```

如果该目录不存在，请手动创建，否则上传功能会报错。

### 2.2 启动

打开项目根目录（`msyy/`），**双击运行**：

```
homestay-booking\start-backend.bat
```

或在终端中：

```bash
cd homestay-booking

# Windows (CMD / PowerShell)
start-backend.bat

# 或直接用 Maven Wrapper
mvnw.cmd spring-boot:run
```

> **首次启动**：Maven Wrapper 会自动下载 Maven 及所有依赖，耗时较长（3-10 分钟），请耐心等待。

### 2.3 验证

当终端出现类似以下日志时，表示后端启动成功：

```
Tomcat started on port(s): 8080 (http)
Started HomestayApplication in X.XXX seconds
```

此时可访问：

| 地址 | 说明 |
|------|------|
| `http://localhost:8080` | 后端根路径 |
| `http://localhost:8080/doc.html` | Knife4j API 文档（Swagger） |

## 第三步：启动前端（Vue 3 + Vite）

### 3.1 安装依赖

```bash
cd homestay-booking-web

# 如果 node_modules 已存在可跳过此步
npm install
```

### 3.2 启动开发服务器

```bash
npm run dev
```

成功后输出：

```
VITE v4.4.9  ready in xxx ms

➜  Local:   http://localhost:3000/
➜  Network: use --host to expose
```

### 3.3 打开浏览器

访问 **`http://localhost:3000`**。

> 前端已配置 Vite 代理，所有 `/api` 和 `/uploads` 请求自动转发到后端 `http://localhost:8080`，开发时无跨域问题。

## 常见问题

### Q1：`mvnw.cmd` 提示找不到 JDK 17

`start-backend.bat` 默认将 `JAVA_HOME` 设为 `C:\Program Files\Java\jdk-17`。

如果你的 JDK 17 在其他路径，请编辑 `start-backend.bat` 第 2 行，改为你的实际路径：

```batch
set JAVA_HOME=D:\你的JDK路径\jdk-17
```

### Q2：MySQL 连接失败 `Communications link failure`

- 确认 MySQL 服务已启动
- 确认数据库 `XRXRXR` 已创建
- 确认已执行 `init.sql` 和 `db-migration.sql`

### Q3：Redis 连接失败 `Unable to connect to Redis`

- 确认 Redis 服务已启动，监听 `6379` 端口
- 确认 Redis 密码为 `204725`（配置文件 `application.yml` 中一致）

### Q4：启动后端报 `upload.path` 目录不存在

手动创建目录：`C:\Users\xr\Desktop\000\1\homestay-booking\uploads`

### Q5：前端 `npm install` 卡住或报错

切换到国内镜像：

```bash
npm config set registry https://registry.npmmirror.com
npm install
```

### Q6：前端页面空白 / API 请求 404

- 确认后端 **8080** 端口已启动
- 不要直接访问 `http://localhost:8080`，要通过前端 `http://localhost:3000` 访问
- Vite 代理仅在开发模式生效，生产部署需 Nginx 反向代理

### Q7：端口被占用

```powershell
# 查看占用 8080 / 3000 的进程
netstat -ano | findstr :8080
netstat -ano | findstr :3000

# 强制结束进程（替换为实际 PID）
taskkill /PID <PID> /F
```
